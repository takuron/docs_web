Git 交互式中文指南

为开发者准备的现代化 Git 命令速查手册

什么是 Git?

Git 是一个免费、开源的**分布式版本控制系统**，旨在快速高效地处理从小型到非常大的项目的所有事务。它用于在软件开发期间跟踪源代码的变化，允许多个开发人员协同工作。本指南汇集了最常用的 Git 命令，帮助您提高开发效率。

交互式命令速查

点击下方按钮筛选命令类别，或展开卡片查看详细说明与示例。

配置用户信息

首次使用 Git 时设置你的用户名和邮箱。

git config --global user.name "Your Name"
详细说明

这是提交代码时署名的依据。`--global` 标志表示这是全局配置。

设置邮箱:
git config --global user.email "you@example.com"

初始化仓库

在当前目录创建一个新的 Git 仓库。

git init

克隆远程仓库

从远程地址下载一个项目的完整历史记录。

git clone <仓库URL>

查看状态

显示工作区和暂存区的状态。

git status
详细说明

这是最常用的命令之一，可以查看哪些文件被修改、哪些文件已暂存，以及当前所在分支。

暂存文件

将文件的更改添加到暂存区，准备提交。

git add <文件名>
详细说明

暂存所有已修改和新文件:
git add .

交互式暂存部分更改:
git add -p

提交更改

将暂存区的内容创建为一次新的提交记录。

git commit -m "提交信息"
详细说明

-m 标志用于直接在命令行提供提交信息。

将已暂存文件追加到上一次提交:
git commit --amend

查看提交历史

显示当前分支的提交历史记录。

git log
详细说明

更美观的单行图形化历史:
git log --graph --oneline --decorate

分支管理

列出、创建或删除分支。

git branch
详细说明
  • 列出所有本地分支: git branch
  • 创建新分支: git branch <分支名>
  • 删除分支: git branch -d <分支名>
  • 强制删除分支: git branch -D <分支名>

切换分支

切换到指定的分支。

git checkout <分支名>
详细说明

创建并立即切换到新分支:
git checkout -b <新分支名>

合并分支

将指定分支的历史合并到当前分支。

git merge <要合并的分支名>

推送更改

将本地分支的提交推送到远程仓库。

git push <远程名> <分支名>
详细说明

通常，远程名是 `origin`。例如:
git push origin main

拉取更改

从远程仓库获取最新版本并合并到本地。

git pull <远程名> <分支名>
详细说明

`git pull` 本质上是 `git fetch` 和 `git merge` 的组合。

重置更改 (暂存区)

取消暂存文件，将其移出暂存区。

git reset HEAD <文件名>

撤销工作区修改

丢弃工作区中对某个文件的所有修改。

git checkout -- <文件名>
详细说明

这是一个危险的操作，会永久丢失工作区的修改，请谨慎使用。

反转提交

创建一个新的提交，内容与指定的旧提交相反。

git revert <提交ID>
详细说明

这是一种安全地“撤销”公开提交的方式，因为它不修改现有历史，而是追加一个新的“反向”提交。

储藏更改

临时保存工作区的修改，让工作区变干净。

git stash
详细说明
  • 查看储藏列表: git stash list
  • 恢复最近的储藏: git stash pop
  • 应用最近的储藏: git stash apply
  • 丢弃最近的储藏: git stash drop
